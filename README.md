# Adamantine AI Gateway

Fail-closed, contract-first gateway for untrusted AI-originated work.

## Overview

Adamantine AI Gateway is a deterministic boundary layer that converts
untrusted AI outputs into structured, validated, policy-controlled
evidence.

External AI\
→ Adapter\
→ AI Gateway\
→ Q-ID → Shield v3 → Adaptive Core → AdamantineOS

## v0.3.0 Highlights

-   Adapter Manifest V1 (contract-driven adapter declarations)
-   AI Gateway Receipt V1 (deterministic evidence artifact)
-   Manifest-aware adapter registry
-   Receipt builder and validation
-   Gateway receipt processing path (`process_with_receipt`)
-   100% test coverage maintained

## Principles

-   Fail-closed always\
-   Contract-first\
-   Deterministic processing\
-   No unknown fields\
-   Canonical-safe payloads\
-   Adapters translate, gateway verifies

## Status

v0.3.0 introduces deterministic evidence foundations for downstream
Adamantine systems.
